Deploying with Ansible

So far, we’ve deployed and configred Skupper and the application manually using skupper and kubectl. One way to automate it is using Ansible.

Ansible

Ansible is an open source IT automation engine that automates provisioning, configuration management, application deployment, orchestration, and many other IT processes. There are different ways to install Ansible. The easiest way is using pip/pipx.

pipx install --include-deps ansible

python3 -m pip install --user ansible

With Ansible installed, we need to install Ansible and Kubernetes Ansible modules.

Installing required Ansible modules

To install Skupper and Kubernetes modules you need to use the ansible-galaxy command already installed with Ansible.

In a terminal window, run the following command:

ansible-galaxy collection install kubernetes.core

ansible-galaxy collection install skupper.network

With modules installed, we can move forward to store the Kubernetes kubeconfig files of noth clusters in the local directory:

Kubeconfigs

To store the kubeconfig of the cluster 1 export KUBECONFIG environment variable to the following value and then run oc login to craete the file with the required values:

export KUBECONFIG=`pwd`/publick8sconfig

oc login --token=sha256~sq_--server=https://api.sandbox-m4.g2pi.p1.openshiftapps.com:6443 (1)
1 Change this to your specific arguments.

Now, we need to store the information of the cluster 2 into another kubeconfig file. First unset the previous KUBECONFIG, switch to kind context, and finally use kind tool to materialize the file:

unset KUBECONFIG

kubectx kind-kind-cluster
kind export kubeconfig --name kind-cluster --kubeconfig `pwd`/privatek8sconfig

At this point, we can refer to these files in Ansible to login to the clusters. Let’s deploy the same application seen on Chapter 2 but using Ansible instead of using CLI commands.

Ansible Manifests

Now, we need to create the two most important files in Ansible, the inventory and the playbook file. In this case, we are going to call the cluster 1 west, and cluster 2 east, as these names makes the playbook look more clear.

Inventory

Create an inventory.yaml file configuring clusters and its relationship with the following content:

inventory.yaml
all:
  vars:
    ansible_connection: local
  hosts:
    west: (1)
      kubeconfig: "{{ inventory_dir }}/publick8sconfig" (2)
      namespace: asotobue-dev (3)
    east: (4)
      kubeconfig: "{{ inventory_dir }}/privatek8sconfig"
      namespace: default
      links: (5)
        - host: west
      services: (6)
        postgres:
          ports:
            - 5432
          targets:
            - type: deployment
              name: postgres
1 Defines the west cluster.
2 Sets the kubeconfig file created in previous section
3 Sets the namespace
4 Defines the east cluster
5 Defines the link between clusters
6 Exposes the postgres service to west

Playbook

The last thing before we can run Ansible is to create the playbook file which does two things, installs the app to both clusters and installs and configures Skupper: Create the following file:

playbook.yaml
- hosts: localhost
  tasks: (1)
    - name: Create To public
      kubernetes.core.k8s: (2)
        src: ./kubernetes/public.yaml
        namespace: asotobue-dev
        kubeconfig: ./publick8sconfig

    - name: Create To private
      kubernetes.core.k8s:
        src: ./kubernetes/private.yaml
        namespace: default
        kubeconfig: ./privatek8sconfig

- hosts: all
  collections:
    - skupper.network (3)
  tasks:
    - import_role:
        name: skupper (4)
1 Apply the application manifests to both clusters
2 Set the deployment file location and the cluster
3 Import Skupper collection
4 Apply the role for installing Skupper

Install the Application

To install the application and the Skupper proxy in both clusters, run the following command:

ansible-playbook -i inventory.yaml playbook.yaml

Clean Up

To clean up the clusters, create a new playbook invoking skupper_delete role.

playbook-delete.yaml
- hosts: all
  collections:
    - skupper.network
  tasks:
    - import_role:
        name: skupper_delete

- hosts: localhost
  tasks:
    - name: Delete public
      kubernetes.core.k8s:
        state: absent
        src: ./kubernetes/public.yaml
        namespace: asotobue-dev
        kubeconfig: ./publick8sconfig

    - name: Delete private
      kubernetes.core.k8s:
        state: absent
        src: ./kubernetes/private.yaml
        namespace: default
        kubeconfig: ./privatek8sconfig
ansible-playbook -i inventory.yaml playbook-delete.yaml
